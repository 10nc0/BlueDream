# Information × Protocol — A Lineage

Five systems. Each adds one layer to the same stack.
Each encodes sovereignty into the structure itself, not into trust.

---

## The Invariant

Every system in this lineage shares three properties:

1. **Information + protocol** — the meaningful unit is inseparable from the rules that move it
2. **Substrate-agnostic** — works on copper wire, fiber, radio, satellite, WhatsApp, LINE, Telegram
3. **Sovereign by design, not by permission** — the guarantee lives in the structure, not the intermediary

---

## The Stack

### TCP/IP — 1974, Cerf & Kahn
**Information:** the packet  
**Protocol:** packets find each other across any network

The sovereignty guarantee is not the wire. It is the packet structure.  
The packet does not ask permission from any single node.

---

### HTTP — 1989, Berners-Lee
**Information:** the document  
**Protocol:** request/response over TCP/IP

The sovereignty guarantee is not the hosting provider. It is the URL.  
Children learn the URL before they understand what it points to.  
The script precedes the literacy. The protocol precedes the comprehension.

No app to install. A browser on a phone, a library terminal, a refrigerator panel — and it works.

---

### Bitcoin — 2008, Satoshi
**Information:** the transaction  
**Protocol:** append-only distributed ledger across any node

The sovereignty guarantee is not the bank. It is the ledger structure.  
Tamper-evident by construction. Any node can verify. No node can revoke.

---

### IPFS — 2015, Benet
**Information:** the content  
**Protocol:** addressed by hash, not by location

The sovereignty upgrade: not *where* the file is — *what* the file is.  
The hash is the address. The address cannot be taken down.  
Content cannot be altered without changing its address.

---

### Nyanbook — 2026
**Information:** the message, the timestamp, the gap  
**Protocol:** webhook → queue → ledger → hash → pin

The sovereignty guarantee is not the Discord thread, not the platform URL.  
It is the SHA256 hash and the IPFS pin.

Substrate-agnostic: WhatsApp, LINE, Telegram — any inpipe,  
anything that can send an HTTP POST.

Built for the $7/day mobile user who never had access to any of the four layers below —  
and now inherits all of them at once.

## The Gap Is the Data

Most archiving systems treat absence as failure — a null, a missing record, an error.

Nyanbook treats the gap as a first-class signal.  
The silence between archive events is itself an observation.  
What wasn't recorded, and when, is part of the record.

This is the insight that connects Nyanbook to its lineage:  
TCP/IP treats lost packets as routing information.  
Bitcoin treats the absence of a transaction as a confirmed state.  
Nyanbook treats the gap in a message archive as a data point.

Information + protocol, all the way down.  
The stack that makes the gap queryable — forever, for the price of electricity.
